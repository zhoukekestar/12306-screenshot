# 12306 适老化截图提取与重绘工具 - 开发计划与 Todo List

## 开发计划 (Plan)

根据 `research.md` 的调研结果，本项目采用纯前端无后端架构，核心是将复杂的 12306 订单截图转换为大字号、高对比度的极简乘车卡片。整体开发计划分为以下几个阶段：

### 阶段一：项目初始化与基础搭建
- 使用 Vite + React 18 + TypeScript 初始化项目架构。
- 配置 Tailwind CSS，确立适老化 UI/UX 设计规范（大字号、高对比度 WCAG AAA 标准、大点击区域）。
- 确立移动优先（Mobile First）的响应式布局策略，确保在手机浏览器与微信环境等移动端操作顺畅。
- 安装核心依赖库：`tesseract.js` (OCR)、`html-to-image` 或 `html2canvas` (截图导出)、`lucide-react` (图标)。

### 阶段二：核心业务逻辑开发
- **图片上传模块**：实现支持点击与拖拽、适配移动端相册选图的超大上传组件。
- **OCR 解析模块**：集成 `tesseract.js`，实现离线中文简体识别，并提供友好的加载进度提示。
- **信息提取模块**：基于正则表达式，从 OCR 文本中精准提取核心要素（车次、时间、日期、座位、车站、姓名）。

### 阶段三：极简结果展示与轻量级修正
- 贯彻“极简操作”理念：用户上传截图解析完成后，直接展示生成好的极简乘车卡片，默认跳过繁琐的核对表单。
- 提供一个轻量级的“修改信息”入口（如辅助按钮或点击卡片文本进入编辑），仅在 OCR 出现极小概率错误时，供用户主动唤起简单的修改面板进行微调。

### 阶段四：适老化卡片渲染与图片生成
- 根据适老化设计规范（关键信息 >= 36px，辅助信息 >= 28px，白底黑字，层级分明），使用 Tailwind 还原结果卡片的排版。
- 集成 `html-to-image`，将渲染好的卡片 DOM 节点转化为可下载的本地图片，提供一键保存/下载功能。
